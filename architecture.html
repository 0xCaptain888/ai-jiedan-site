Technology Selection — 每一层技术栈及选型理由

层级 技术 选型理由
前端Next.js 14 + TypeScript + TailwindCSSSSR/SSG性能优化，SEO友好，组件化开发效率高
移动端React Native / Taro (微信小程序)跨平台复用前端能力，Taro原生支持微信生态
API网关Kong限流、鉴权、日志、插件生态丰富，高性能OpenResty底层
后端框架Go (Gin) + Python (FastAPI)Go处理高并发业务逻辑，Python处理AI相关服务，各取所长
微服务通信gRPC + Kafka同步调用用gRPC（低延迟、强类型），异步解耦用Kafka消息队列
主数据库PostgreSQL 16ACID事务保障，JSON/JSONB原生支持，扩展性强，社区成熟
缓存Redis 7热数据缓存、分布式锁、排行榜、会话管理，毫秒级响应
搜索引擎Elasticsearch 8全文检索，服务者/需求搜索，支持中文分词，聚合分析强
文件存储MinIO / Alibaba Cloud OSSS3协议兼容，成本可控，私有化部署灵活
向量数据库MilvusAgent知识库存储，语义检索，高性能ANN索引
LLM调用通义千问 + Claude API国内场景用通义千问（合规/低延迟），复杂任务用Claude（推理强）
Agent框架LangChain + LangGraph工作流编排、工具调用、状态管理，社区生态活跃
容器编排Docker + Kubernetes弹性伸缩、滚动更新、服务发现、自愈能力
CI/CDGitHub Actions与代码仓库深度集成，自动化测试部署，YAML配置简洁
监控Prometheus + Grafana指标采集+可视化告警，K8s原生集成，开源免费

核心API接口设计

RESTful API Endpoints — 按服务模块划分

用户服务 User Service

POST /api/v1/auth/sms-code 发送验证码
POST /api/v1/auth/login 手机号登录
GET /api/v1/users/me 获取当前用户信息
PUT /api/v1/users/me 更新用户信息
GET /api/v1/providers/:id 获取服务者详情

需求服务 Demand Service

POST /api/v1/demands 创建需求
POST /api/v1/demands/:id/parse AI解析需求
GET /api/v1/demands/:id/matches 获取匹配服务者
PUT /api/v1/demands/:id 更新需求

订单服务 Order Service

POST /api/v1/orders 创建订单
GET /api/v1/orders/:id 获取订单详情
PUT /api/v1/orders/:id/accept 接单
PUT /api/v1/orders/:id/deliver 提交交付
PUT /api/v1/orders/:id/complete 确认完成
GET /api/v1/orders/:id/timeline 获取订单时间线

Agent服务 Agent Builder Service

POST /api/v1/agents 创建Agent
GET /api/v1/agents/:id 获取Agent详情
POST /api/v1/agents/:id/test 测试Agent
PUT /api/v1/agents/:id/deploy 部署Agent

评价服务 Review Service

POST /api/v1/reviews 提交评价
GET /api/v1/providers/:id/reviews 获取服务者评价

弹性伸缩策略 Auto-Scaling Policies

HPA (Horizontal Pod Autoscaler)
CPU > 70% → 扩容 (max 20 replicas)
CPU < 30% → 缩容 (min 2 replicas)
QPS > 1000/pod → 触发扩容
Memory > 80% → 告警 + 扩容
关键SLA指标
API P99 延迟 < 200ms
可用性 > 99.95%
RTO < 5min / RPO < 1min
数据库自动备份: 每6小时 + binlog
CI/CD Pipeline
1. Push to main → 触发 GitHub Actions
2. 单元测试 + 集成测试 + Lint
3. Docker Build → 推送到 ACR 镜像仓库
4. Staging 自动部署 → 人工验证
5. 审批后 → Production 滚动更新
监控告警
Prometheus 采集所有服务指标
Grafana 可视化Dashboard
ELK 集中式日志分析
告警渠道: 钉钉/企微/短信/邮件
